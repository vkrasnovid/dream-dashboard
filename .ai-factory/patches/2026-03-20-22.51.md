# Fix QA Issues: M-1, M-2, L-1..L-5 in Dream Dashboard

**Date:** 2026-03-20 22:51
**Files:** index.html, QA_REPORT.md
**Severity:** medium

## Problem

Seven issues found by QA in index.html:
- M-1: Horizontal scroll at 320px viewport (minmax too wide)
- M-2: Nav highlight flickers when two sections simultaneously visible
- L-1: Tooltip uses hard-coded `#0f172a` instead of `var(--bg)`
- L-2: `relativeTime()` returns negative values for future timestamps
- L-3: Six `console.log` calls emit on every page load and filter click
- L-4: Sidebar overlay disappears abruptly (display:none kills transition)
- L-5: `lang="en"` on page with Russian as primary language

## Root Cause

- M-1: `minmax(320px, 1fr)` ignores container width constraints below 320px.
- M-2: IntersectionObserver processes entries individually; last-wins is non-deterministic when two sections are ≥10% visible simultaneously.
- L-1: Hard-coded hex value used instead of the existing CSS variable `--bg`.
- L-2: No guard for negative diff (future timestamps or timezone offsets).
- L-3: Debug logs left in production code path — no flag to suppress them.
- L-4: `display: none` is applied immediately when `.visible` is removed, so the opacity transition has no DOM element to animate.
- L-5: `<html lang="en">` misidentifies the primary Russian text for screen readers.

## Solution

- M-1: Changed to `minmax(min(320px, 100%), 1fr)` — respects container width.
- M-2: Store visibility state in `dataset.visible`; after all entries fire, pick the topmost visible section by `getBoundingClientRect().top` — deterministic, flicker-free.
- L-1: Replaced `#0f172a` with `var(--bg)`.
- L-2: Added `if (diff < 0) return 'just now';` as first check in `relativeTime()`.
- L-3: Added `const DEBUG = location.search.includes('debug');` and gated all 6 `console.log` calls behind it.
- L-4: Changed overlay to always `display: block`; added `pointer-events: none` when not visible, `pointer-events: auto` when visible — opacity transition now plays on both open and close.
- L-5: Changed `<html lang="en">` to `<html lang="ru">` since Russian is the primary language in all bilingual labels.

## Prevention

- For grid responsive layouts: always use `min(fixedPx, 100%)` in minmax when fixed px could exceed container at narrow viewports.
- For IntersectionObserver scroll-spy: never process entries individually for exclusive active state — always re-evaluate all observed elements after each batch.
- Use CSS variables consistently; avoid duplicating literal values that already have a variable.
- Future-proof time display functions with a guard for negative diffs.
- Gate all diagnostic logs behind a debug flag before shipping.
- For CSS transitions on elements with display: none — use opacity + pointer-events instead.
