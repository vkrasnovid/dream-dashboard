# QA batch fix: icon, sort, hover, JSON safety, max-width, nav threshold

**Date:** 2026-03-20 22:40
**Files:** index.html
**Severity:** high (2), medium (5)

## Problem

Seven QA issues identified in `QA_REPORT.md`:
- H-1: `fa-magnifying-glass-chart` (FA Pro) rendered blank for "review" feed rows
- H-2: Activity feed rendered in insertion order — not newest-first
- M-1: Stats section heading already present (no-op)
- M-2: Nav IntersectionObserver threshold 0.2 caused active-link gap between sections
- M-3: `.feed-row` had `transition: background` but no `:hover` rule — transition was dead
- M-4: Three bare `JSON.parse()` calls at top level — any SyntaxError blanked the page
- M-5: `.main` had no `max-width` — content stretched beyond usable width on 1920px screens

## Root Cause

- H-1: FA Pro icon name used in a project loading FA Free CDN.
- H-2: Feed was rendered directly from JSON insertion order with no sort step.
- M-2: 40% threshold too high — scrolling between tall sections left no section intersecting at that level.
- M-3: CSS transition declared without a matching `:hover` rule — common authoring oversight.
- M-4: Bare `const x = JSON.parse(...)` at module top-level; any malformed JSON propagates and aborts the entire `DOMContentLoaded` handler.
- M-5: `flex: 1` with no `max-width` on `.main` lets content expand to viewport on ultrawide monitors.

## Solution

- H-1: Changed `typeIcon.review` to `'fa-eye'` (FA Free solid).
- H-2: Added `.slice().sort((a, b) => new Date(b.timestamp) - new Date(a.timestamp))` inside `renderFeed()` before mapping to HTML.
- M-2: Lowered `threshold` from `0.2` to `0.1`.
- M-3: Added `.feed-row:hover { background: var(--surface-alt); }` after the `.feed-row` rule.
- M-4: Replaced bare `const` declarations with `let` + individual `try/catch` blocks, each logging `[FIX]` on error and falling back to `[]`.
- M-5: Added `max-width: 1400px; margin: 0 auto;` to `.main`.

## Prevention

- Always verify FA icon names against the FA6 Free icon list before committing.
- When rendering time-ordered feeds, add an explicit `.sort()` step — never rely on data source ordering.
- IntersectionObserver thresholds for nav highlight should stay at ≤ 0.1 to avoid gap states.
- Pair every CSS `transition` with the corresponding state rule (`:hover`, `.active`, etc.) in the same CSS block.
- Wrap all `JSON.parse` calls at module initialization in `try/catch` with a safe fallback value.
- Apply `max-width` + `margin: auto` to main content containers for wide-screen layouts.
