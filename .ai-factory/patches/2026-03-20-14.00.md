# Fix HIGH/MEDIUM QA issues in Dream Team Dashboard

**Date:** 2026-03-20 14:00
**Files:** index.html
**Severity:** high

## Problem

Five issues found by QA in the static single-file dashboard:

1. **HIGH** — `fa-magnifying-glass-chart` (FA Pro) mapped to `review` activity type; icon rendered as empty box on all 3 review rows.
2. **HIGH** — Activity feed rendered in JSON insertion order; within-day items went oldest-first, and cross-day ordering was incorrect.
3. **MEDIUM** — `.feed-row` declared `transition: background 100ms ease` but no `:hover` rule existed; hover effect was dead.
4. **MEDIUM** — Three bare `JSON.parse()` calls at bootstrap level; any SyntaxError blanked the entire page with no feedback.
5. **MEDIUM** — `.main` had no `max-width`; on 1920px monitors content stretched excessively.

## Root Cause

1. Icon name copied from FA Pro docs without verifying it exists in FA6 Free.
2. `renderFeed()` passed the raw filtered array to `.map()` without sorting first.
3. Transition was added but the matching `:hover` rule was never written.
4. JSON was parsed inline at `const` declaration level — any parse failure threw before any catch could run.
5. Layout was designed for laptop widths; ultra-wide breakpoint not considered.

## Solution

1. Replaced `fa-magnifying-glass-chart` with `fa-eye` in `typeIcon` map (line 975).
2. Chained `.slice().sort((a, b) => new Date(b.timestamp) - new Date(a.timestamp))` before `.map()` in `renderFeed()` (line 982).
3. Added `.feed-row:hover { background: var(--surface-alt); }` after `.feed-row` block (line 457).
4. Converted `const` declarations to `let` with individual `try/catch` blocks; errors logged with `[FIX]` prefix, fallback to `[]` (lines 797–802).
5. Added `max-width: 1400px; margin: 0 auto;` to `.main` CSS rule (lines 130–131).

## Prevention

- Always verify FA icon names against the FA6 Free icon search before using them.
- Sort data before rendering whenever insertion order cannot be guaranteed.
- When adding a CSS `transition`, immediately write the paired state rule (`:hover`, `.active`, etc.).
- Wrap all `JSON.parse` calls in try-catch, especially at bootstrap level where a failure would silence the entire app.
- Add a wide-viewport breakpoint check when designing fluid layouts.
